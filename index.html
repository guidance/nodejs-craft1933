Craft 1933

The Use Case

  • Flash Sales Site
  • User has 10 minutes to complete purchase.
  • If user doesn't purchase, items are removed from cart.
  • Timeout and cleanup should happen even if user leaves session.

The Set Up

  • Magento is already validating the quote on each page load.
  • Active browsing no problemo.

The Problem

  • Users abandoning the session.
  • No page load... no cart validation.
  • Need a cleanup process!

The Problem

  • "I need a cup of coffee."
  • "Mmmm.. wine. Great idea!"
  • "Squirrel!"

Possible Solutions

Polling Cart Items

  • Query quote items table for expired items.
  • Timeout if old quote found.

Polling Cart Items

Advantages

  • The "traditional" way
  • Easy - just set up a cron.

Polling Cart Items

Disadvantages

  • Will never be real time
  • Resource Intensive
  • "I feel dirty"

Event Driven Model with Node.js

Advantages

  • Real time
  • Very lightweight
  • Familiar. We know JavaScript
  • Bling! Bling!

Event Driven Model with Node.js

Disadvantages

  • Fledgling Technology
  • Documentation?
  • New service to support

Decisions

  • Eventing is the better way
  • Progressive client
  • Not a "mission-critical" service.

Procrastinaor

Procrastinator

  • Delay Queue
  • VERY simple service
  • Send it a message with a timeout value
  • It will callback when time's up.
  • No external business knowledge.

API

  • RESTFUL
  • Simple Endpoints, HTTP verbs

POST http://127.0.0.1:8000/items content-type: application/json

{ "callbackUrl": "http://127.0.0.1:8008/callback/item/{{id}}", "interval" : 5000, "data" : {"e":1, "a":0} }

In Action

  • User adds to cart.
  • Magento handler sends message to Node.
  • Node creates async timeout with callback URL.
  • Times Up!
  • Node pings callback url with data packet.
  • Magento handles callback.
